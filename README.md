# Parser Inomarker

Сервис для обхода сайтов, поиска вхождений по регулярным выражениям из `db.json`, админки и отчетов.

## Архитектура
- `php/`:
  - админка (`public/index.php`)
  - фоновые задачи (`bin/console worker:run`)
  - БД SQLite, репозитории, сервис матчинга regex
- `crawler/`:
  - Node.js + Puppeteer Stealth API (`POST /crawl`)
- `docker-compose.yml`:
  - `app` (веб-админка)
  - `worker` (фоновый сканер)
  - `crawler` (headless обход страниц)

## Что умеет админка
- Добавление сайтов для мониторинга.
- Отдельные страницы:
  - `Dashboard` (сводка, последние запуски и вхождения)
  - `Sites` (каталог сайтов, управление, переход к редактированию и отчету)
  - `Settings` (настройки воркера и crawler)
- Настройка фонового скана:
  - количество сайтов за цикл (`worker_batch_size`)
  - интервал автоскана (`scan_interval_minutes`)
  - лимит страниц и глубина обхода
- Отдельные страницы добавления и редактирования сайта.
- Удаление сайта с подтверждением (нужно ввести точное имя сайта).
- Ручной запуск сканирования сайта, пауза/возобновление и отмена текущего запуска.
- Полная очистка данных по сайту при удалении (сайт, запуски, страницы, вхождения).
- Полноценный отчет по конкретному сайту (`/sites/{id}`):
  - сводка запусков и статусов
  - сводка проиндексированных страниц и HTTP ошибок
  - топ сущностей и последние вхождения
  - история запусков и последние страницы
- Retry crawler-запроса (несколько попыток в рамках одного запуска).
- Автопауза сайта при окончательной ошибке, чтобы избежать повторяющихся падений в логе.
- Защита от дублей сайтов по `base_url`.
- Настройка паузы между переходами страниц (`crawler_page_pause_ms`, по умолчанию 1000 мс).
- Страницы с найденными вхождениями помечаются и больше не переобрабатываются, чтобы не дублировать находки.
- Просмотр:
  - последних запусков
  - найденных вхождений (сайт, URL, категория, сущность, фрагмент).

## Быстрый старт
```bash
docker compose up --build
```

После запуска:
- админка: `http://localhost:8080`
- crawler health: `http://localhost:3000/health`
- вход в админку: пароль по умолчанию `лох` (можно переопределить через `ADMIN_SECRET_PASSWORD`)

Сессия в админке хранится 24 часа после успешного входа.

## CLI команды
В контейнере `app` или `worker`:
```bash
php bin/console migrate
php bin/console worker:run --once
php bin/console scan:site <site_id>
```

## Тесты
```bash
cd php
composer install
composer test
```

## Примечания по скрытности crawler
- Используется `puppeteer-extra-plugin-stealth`.
- Рандомизируются `user-agent`, viewport и задержки между страницами.
- Скрываются типовые automation-признаки (`navigator.webdriver` и related fingerprints).

## Нагрузка на CPU (особенно для 1 ядра)
- В `Settings` выставьте `Параллельных сканов` = `1`.
- Увеличьте `Пауза между переходами страниц (мс)` до `1500-3000`.
- В `docker-compose` можно усилить щадящий режим через:
  - `CRAWLER_DYNAMIC_SCROLL_ENABLED=0`
  - `CRAWLER_RESOURCE_BLOCKING_ENABLED=1`
  - `CRAWLER_BLOCK_RESOURCE_TYPES=image,media,font,stylesheet`
  - `CRAWLER_PROGRESS_MIN_LEVEL=info`
  - `SITE_PROGRESS_MIN_LEVEL=info`
